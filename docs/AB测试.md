# A/B测试

## 什么是A/B测试？

A/B测试是一种有效的精细化运营手段，用于提升App产品转化率、优化获客成本。

在对产品进行A/B测试时，我们可以为同一个优化目标（例如优化订阅率）制定两个方案（例如两个内购页），让一部分用户使用A方案，另一部分用户使用B方案，统计对比不同方案的转化率、点击率、留存率等指标，以判断不同方案的优劣并进行决策，从而提升转化率。

## 怎样在Dingyue建立一个A/B测试？

首先，我们可以建立一个测试组，测试对象可以是用户群、内购页以及开关。

当测试对象为用户群时，我们可以选择多个用户群体（注：用户群体在”用户管理“创建），但只能绑定同一个内购页（注：内购页在“内购页”创建）以及相同的开关，这是为了控制变量，如果您的用户群已经绑定其他内购页也不用担心，A/B测试的权重最高，将覆盖原有内购页，当您关闭A/B测试时，内购页恢复正常。

![选择对象](/images/32.png)

当您选择完毕后点击“下一步”，为您的测试选择易于记忆的名称，填写测试组描述以及计划，如果您想立即开始测试，则勾选“开启测试组”，然后点击“创建测试组”。您的A/B测试组创建完成。

![创建测试组](/images/33.png)

创建完成后，您可以在本页面查看下载测试数据。请注意，您可以建立多个A/B测试组，但只能有一个测试组开启并进行测试。
